### 2-1. 성장 시스템(BM제외)

엔드필드의 성장 시스템은 크게 2개로 구성된다

1. 캐릭터 육성
 - 캐릭터 레벨업
 - 스킬 강화
 - 장비 착용

2. 무기 육성
 - 무기 레벨업
 - 기질 착용

각 시스템은 UI상 독립적으로 보여지나, 최종 데미지 계산에 누적 반영되는 것으로 보인다

### 2-1-1 캐릭터 육성

1. 캐릭터 레벨업
 - 2개의 구간 구조 (1~80 / 81~90)
 - 구간별 소모 경험치 재화 상이
 - 특정 구간마다 추가 재화 요구 발생

 ! 기획 의도 분석
 - 80레벨이 1차 성장 종착점으로 설계했을 가능성
 - 후반 구간은 초반 구간과의 육성 재화 분리로 성장 컨텐츠 장기 체류 목적
 - 추가 재화는 경험치 던전과 별개 컨텐츠 요구로 추가 체류 목적

 ? 테스트 포인트
 - 1~80 / 81~90 구간별 경험치 요구량 그래프로 재화 소모량 정상 유무 확인
 - 레벨업시 재화 소모 로직 검증
    - 필요 이상 투입시 재화 반환 여부   
    - 부족시 UI 안내 정상 노출 여부
 - 특정 구간 진입시 레벨업 잠금, 추가 재화 요구 정상 작동 확인
 - 서버 저장, 재접속 시 동기화 여부 확인



2. 스킬 강화
 - 4개의 스킬 강화 구성 (기본 공격, 배틀 스킬, 연계 스킬, 궁극기)
 - 각 스킬별로 기본 1레벨에서 일반 강화 9회 진행, 이후 특수 강화 3회 진행
 - 3개의 구간 구조이며 캐릭터 특정 레벨 구간에 가능한 스킬업 한계가 존재
 - 각 구간별마다 강화 배율이 높아지는만큼 추가 재화를 요구함

 ! 기획 의도 분석
 - 10레벨이 1차 성장 종착점으로 설계했을 가능성
 - 구간별 육성 재화 분리로 성장 컨텐츠 장기 체류 목적
 - 특수 강화에서 요구하는 한정 재화는 유저에게 까다로운 선택을 강요

 ? 테스트 포인트
 - 각 스킬 강화 단계별 요구 재화 정확성 및 차감 로직 검증
 - 재화 부족시 UI 안내 정상 노출 여부
 - 캐릭터 레벨 조건에 따른 강화 제한 정상 작동 확인
 - 강화 수치가 실제 전투 데미지 배율에 정상 반영되는지 확인
 - 서버 저장, 재접속 시 동기화 여부 확인



3. 장비 착용
 - 인게임 공장 및 탐험 컨텐츠로 장비의 습득 가능
 - 캐릭터 레벨별로 착용할 수 있는 장비의 제한이 있다
 - 각 장비들은 개별 능력치, 세트 효과가 있으며 이는 캐릭터의 능력치와 데미지 배율, 전투 사이클에 직접적인 영향을 준다
 - 특정 장비는 특정 요구 장비와 공장 컨텐츠 재화를 소모하는 단조로 능력치를 3번 향상시킬 수 있으며, 단계별로 비공개된 확률이 적용된다
 - 일정 단조 횟수 이후 확정 단조 시스템이 활성화된다

 ! 기획 의도 분석
 - 확률 컨텐츠로 중복 파밍이 요구되며 컨텐츠의 장기 체류 목적
 - 각 캐릭터별 요구 능력치의 세분화만큼 다양한 장비들로 세팅 다양화 의도로 파밍의 재미 유도
 - 장비 단조라는 3단계 능력치 고정 향상과 확정 단조 시스템으로 유저의 성장 격차 최소화를 노림

 ? 테스트 포인트
 - 장비 단조 로직 정상 작동 확인
    - 단조시 선택한 재화의 정상 소모 여부
    - 단조 성공시 확정 단조 누적 카운트 초기화 여부
    - 확정 단조 누적 수치 이후 다음 단조가 100% 성공 하는지 확인
 - 캐릭터 장착시 개별 장비 능력치 반영 및 세트 효과 활성화 확인 //UI로 보여지는지 확인 필요
 - 서버 저장, 재접속 시 동기화 여부 확인

2. 기질 착용 및 식각

 - 무기가 갖고있는 능력치와 동일한 능력치가 착용한 기질에 있을경우 그 능력치의 레벨만큼 무기 능력치 레벨이 증가된다
 - 기질은 같은 등급의 다른 기질을 사용하여 능력치를 강화할 수 있다
    - 장비 강화와 같이 기질은 식각이라는 시스템이 있으며, 일정 확률이 존재하며 확정 보정 시스템이 존재한다
    - 기본 무기의 스킬 상태가 3/3/1일 경우 모든 옵션이 최고 레벨 상태로 강화된 기질을 착용할 시 9/9/4가 된다   

 ! 기획 의도 분석

 ? 테스트 포인트
 - 기질 착용 시 동일 능력치가 정상 합산 되는지 확인
 - 동일 능력치 없는 기질 장착 시 변화 확인   
 - 실제 전투 시 증가된 능력치가 정상 적용 되는지 확인
 - 식각 시 강화 로직 검증
    - 식각시 재료로 사용한 기질의 정상 소모 여부
    - 식각 성공시 확정 식각 누적 카운트 초기화 여부
    - 확정 식각 누적 수치 이후 다음 식각이 100% 성공 하는지 확인

### 2-2. 테스트 관점 정리

1. 기능 정확성
 - 성장 시스템 진행 시 요구(선택) 재화 및 재료가 정상 소모하는지
 - 능력치 레벨 상승에 따른 세부 수치가 정상 반영하는지
 - 성장 시스템 적용 후 실제 전투 시 능력치 및 데미지 계산이 정상 작동하는지

2. UX
 - 성장 시스템 진행 전 레벨이나 재화에 따른 제한 부분에 대한 피드백이 되는지
 - 성장 시스템 진행 시 사용할(선택한) 재화 및 재료에 대한 확인이 되는지
 - 성장 시스템 진행 후 재화 소모 및 결과 피드백이 명확히 전달되는지

3. 데이터 안정성
 - 각 성장 시스템 진행 후 결과 데이터가 서버와 정상 동기화 하는지
 - 서버 연결 끊어짐 혹은 클라이언트 강제 종료에 대응 되는지

### 2-3. 예상 버그 유형

1. 기질 교체 시 능력치가 정상 갱신되지 않는 문제

 - 재현 절차
    1. 능력치 1레벨 옵션을 가진 기질 장착 확인
    2. 해당 기질 장착 해제
    3. 능력치 4레벨 옵션을 가진 다른 기질 장착
    4. 능력치 창 확인

 - 기대 결과
    - 무기 능력치 레벨 3, 기질 능력치 레벨 4로 합산 능력치 레벨이 7로 표시되어야 한다

 - 실제 결과
    - 능력치 레벨이 7로 합산되지 않고 기존 기질 기준인 레벨 4로 표시된다

2. 확정 단조 카운트 도달 후 성공하지 않는 문제

 - 재현 절차
    1. 장비 단조 실패를 반복하여 확정 단조 누적 카운트에 도달
    2. 확정 단조 조건이 충족된 상태에서 장비 단조 진행
    3. 장비 능력치 확인

 - 기대 결과
    - 확정 보정 시스템이 작동하여 장비는 100% 단조되고 누적 카운트는 초기화 되어야 한다.

 - 실제 결과
    - 확정 단조 조건이 충족되었음에도 단조는 실패하고 누적 카운트만 초기화되었다.

3. 레벨업 진행 중 서버 종료 후 재접속 시 소모된 재화와 캐릭터 레벨 데이터의 불일치 문제

 - 재현 절차
    1. 캐릭터 레벨업 정상 진행
    2. 레벨업 처리 중 서버 강제 종료
    3. 재접속 후 캐릭터 상태 확인

 - 기대 결과
    - 레벨업 데이터의 정상 저장으로 재접속 시 캐릭터 레벨이 상승한 상태여야 함

 - 실제 결과
    - 레벨업에 사용된 재화는 소모되었으나 캐릭터 레벨업은 상승되지 않은 상태로 유지되었다

### 2-4. QA 관점 개선 제안

1. 기질 식각 시도 이후 하단의 기질 목록 UI가 매번 무작위 재정렬되어,
   유저가 소모를 원하는 기질을 재탐색 해야하는 불편함이 발생

 - 개선 제안
    1. 서버 데이터를 통한 기질 획득 순 나열
    2. 기질 잠금 기능을 활용하여 유저가 잠금한 기질은 기질 목록에서 제외하는 필터 기능 추가

- 기대 효과
    - 반복적인 기질 탐색 과정이 줄어들어 시스템 사용 편의성이 향상된다

2. 스킬 강화 UI에서 캐릭터 레벨로 의해 강화가 제한되는 스킬 레벨 구간이 명확히 표기되지 않아
   유저가 강화 가능 여부를 직관적으로 파악하기 힘들다

 - 개선 제안
    1. 레벨 제한이 존재하는 스킬 레벨 구간을 색상, 구분선을 통해 시각적으로 구분이 갈 수 있게 한다
    2. 스킬 레벨 클릭시 스킬 UI 상단에 요구 레벨을 표기한다

 - 기대 효과
    1. 유저가 스킬 강화 여부를 직관적으로 파악을 할 수 있게 되어 육성 시스템의 선택과 집중이 가능해짐
    2. 게임 내 정보로 외부 사이트의 의존도 감소 효과

3. 단조 혹은 식각 시도 후 결과가 별도의 UI로 표시 되지 않아
   유저가 장비 상황의 탐색 시간을 항상 가져야 하는 불편이 발생

 - 개선 제안
    - 강화 시스템 시도 후 결과를 별도의 UI 팝업으로 표시하여
      성공 여부 및 변경 능력치, 보정 시스템 카운트를 즉시 확인할 수 있게 한다

 - 기대 효과
    - 유저의 반복적인 탐색 시간의 감소로 편의성 개선
    - 별도 강화 UI로 시스템 사용 흐름의 유연성 증진
